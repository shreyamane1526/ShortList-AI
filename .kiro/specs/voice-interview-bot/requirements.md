# Requirements Document

## Introduction

The Voice Interview Bot is a browser-based AI interview feature integrated into the candidate side of the recruiter/candidate web application. When a candidate views their evaluation detail page, they can launch a full-screen modal that conducts a structured 5-question voice interview. The AI speaks questions aloud using the browser's Text-to-Speech API, the candidate responds using the browser's Speech-to-Text API, and each answer is assessed by the Groq LLM. After all five questions, a transcript and AI assessment report are displayed and saved to the backend. The feature degrades gracefully to a text-based flow on browsers that do not support the Web Speech API.

---

## Glossary

- **VoiceInterviewModal**: The full-screen modal overlay component that hosts the entire interview experience.
- **InterviewStateMachine**: The `useReducer`-based state machine managing interview lifecycle transitions.
- **SpeechRecognitionHook**: The `useSpeechRecognition` custom hook wrapping the browser Web Speech API.
- **TextToSpeechHook**: The `useTextToSpeech` custom hook wrapping the browser `speechSynthesis` API.
- **InterviewHook**: The `useInterview` custom hook that owns the top-level interview state machine.
- **QuestionSet**: An ordered array of exactly 5 objects, each with `id`, `question`, and `focus_area` fields, generated by the Groq LLM.
- **AnswerAssessment**: A per-question LLM evaluation object containing `score` (1–10), `sentiment`, `key_point`, and optional `follow_up_hint`.
- **Transcript**: The complete record of all questions, candidate answers, and assessments for a single interview session.
- **CandidateInterview**: The SQLAlchemy model and database table that persists a completed interview session.
- **GroqAPI**: The existing `POST /api/chat/ask` backend endpoint that proxies requests to the Groq LLM.
- **EvaluationContext**: The combination of evaluation summary, job title, and candidate skill gaps used to generate interview questions.
- **PhaseIntro**: The first visual phase of the modal showing session metadata and a "Begin Interview" button.
- **PhaseActive**: The second visual phase where the live interview takes place.
- **PhaseResults**: The third visual phase showing the completed transcript, scores, and action buttons.
- **MicButton**: The large microphone toggle button shown during PhaseActive.
- **AIAvatar**: The animated SVG circle component that pulses when the AI is speaking.
- **TextFallback**: The typed-answer flow shown on browsers that do not support the Web Speech API.

## Requirements

### Requirement 1: Entry Point

**User Story:** As a candidate, I want a clearly visible button on my evaluation detail page, so that I can launch the AI voice interview without navigating away.

#### Acceptance Criteria

1. THE VoiceInterviewModal SHALL be accessible via a "Start AI Interview (Voice)" button rendered on the candidate evaluation detail page at route `/candidate/evaluations/<eval_id>`.
2. WHEN the candidate clicks the "Start AI Interview (Voice)" button, THE VoiceInterviewModal SHALL open as a full-screen modal overlay without navigating to a new page.
3. WHEN a completed CandidateInterview record exists for the current evaluation, THE evaluation detail page SHALL display a "You completed an interview — view results" notice in place of the "Start AI Interview (Voice)" button.

---

### Requirement 2: Browser Compatibility Detection

**User Story:** As a candidate using an unsupported browser, I want a clear fallback message and alternative, so that I am not blocked from completing the interview.

#### Acceptance Criteria

1. WHEN the VoiceInterviewModal initialises, THE InterviewStateMachine SHALL check for the presence of `window.SpeechRecognition` or `window.webkitSpeechRecognition`.
2. IF the Web Speech API is not detected, THEN THE VoiceInterviewModal SHALL display a banner stating "Voice interview requires Chrome or Edge. You can still proceed with a text-based interview."
3. WHERE the TextFallback is active, THE VoiceInterviewModal SHALL replace all microphone input controls with a text input field while preserving the same question flow, progress tracking, and assessment logic.

---

### Requirement 3: Question Generation

**User Story:** As a candidate, I want the AI to generate interview questions tailored to my job evaluation, so that the interview is relevant to the specific role I applied for.

#### Acceptance Criteria

1. WHEN the candidate clicks "Begin Interview" in PhaseIntro, THE InterviewHook SHALL call `POST /api/chat/ask` with a prompt requesting exactly 5 interview questions as a JSON array of `{id, question, focus_area}` objects, incorporating the evaluation summary, job title, and candidate skill gaps as context.
2. WHEN the Groq API returns a valid JSON array of 5 questions, THE InterviewStateMachine SHALL transition from `loading_questions` to `intro` and store the QuestionSet in component state.
3. IF the Groq API call fails or returns malformed JSON during question generation, THEN THE VoiceInterviewModal SHALL display an error state with a "Retry" button that re-triggers question generation.
4. THE InterviewHook SHALL parse the LLM response as JSON and validate that it is an array of exactly 5 objects each containing `id`, `question`, and `focus_area` fields before storing the QuestionSet.

---

### Requirement 4: Interview State Machine

**User Story:** As a candidate, I want the interview to progress through well-defined stages, so that I always know what is expected of me at each step.

#### Acceptance Criteria

1. THE InterviewStateMachine SHALL manage state using `useReducer` and SHALL support the following ordered states: `idle` → `loading_questions` → `intro` → `speaking_question` → `waiting_for_answer` → `recording` → `processing_answer` → `speaking_question` (next question) or `completed`.
2. WHEN all 5 questions have been answered and assessed, THE InterviewStateMachine SHALL transition to `completed` and render PhaseResults.
3. WHEN the candidate presses the Escape key during PhaseActive, THE VoiceInterviewModal SHALL display a pause modal offering "Resume" and "Exit Interview" options.
4. THE InterviewStateMachine SHALL track the current question index (0–4), the elapsed duration in seconds, and the accumulated Transcript entries.

---

### Requirement 5: Text-to-Speech (AI Speaking)

**User Story:** As a candidate, I want to hear the interview questions spoken aloud, so that I can focus on formulating my answer rather than reading.

#### Acceptance Criteria

1. WHEN THE InterviewStateMachine enters `speaking_question`, THE TextToSpeechHook SHALL invoke `window.speechSynthesis.speak()` with the current question text.
2. THE TextToSpeechHook SHALL select a voice where `voice.lang === 'en-US'` and `voice.name` includes 'Google' or 'Natural' if such a voice is available, and SHALL fall back to the browser default voice otherwise.
3. THE TextToSpeechHook SHALL set `SpeechSynthesisUtterance.rate` to `0.92` and `SpeechSynthesisUtterance.pitch` to `1.0`.
4. WHILE THE InterviewStateMachine is in `speaking_question`, THE VoiceInterviewModal SHALL display the current question text as subtitles below the AIAvatar.
5. WHILE THE InterviewStateMachine is in `speaking_question`, THE VoiceInterviewModal SHALL display a "Replay question" button that re-invokes `window.speechSynthesis.speak()` with the same question text.
6. WHEN `window.speechSynthesis` finishes speaking, THE InterviewStateMachine SHALL transition to `waiting_for_answer`.

---

### Requirement 6: Speech-to-Text (Candidate Answering)

**User Story:** As a candidate, I want my spoken answer transcribed in real time, so that I can verify the system has captured my response accurately before submitting.

#### Acceptance Criteria

1. WHEN THE InterviewStateMachine enters `waiting_for_answer`, THE VoiceInterviewModal SHALL display a large MicButton and a "Start Answer" button.
2. WHEN the candidate clicks "Start Answer" or presses the Space key, THE InterviewStateMachine SHALL transition to `recording` and THE SpeechRecognitionHook SHALL start a `SpeechRecognition` instance with `continuous` set to `false` and `interimResults` set to `true`.
3. WHILE THE InterviewStateMachine is in `recording`, THE VoiceInterviewModal SHALL display a live transcript of the candidate's speech as interim results arrive.
4. WHILE THE InterviewStateMachine is in `recording`, THE MicButton SHALL render with a red background and a CSS keyframe pulse animation on a surrounding circle element.
5. WHILE THE InterviewStateMachine is in `recording`, THE VoiceInterviewModal SHALL display a "Listening…" status label.
6. WHEN the candidate clicks "Submit Answer" or presses the Enter key, THE InterviewStateMachine SHALL transition to `processing_answer` using the final transcribed text.
7. WHEN 3 seconds of silence are detected by the SpeechRecognitionHook, THE InterviewStateMachine SHALL automatically transition to `processing_answer` using the accumulated transcript.
8. IF the SpeechRecognitionHook fires an error event during `recording`, THEN THE VoiceInterviewModal SHALL display "We lost audio — please re-record" with a re-record button that restarts recognition while preserving any partial transcript already captured.
9. IF the candidate denies microphone permission, THEN THE VoiceInterviewModal SHALL display clear instructions with a browser icon and a "Use text instead" button that activates the TextFallback.

---

### Requirement 7: Answer Assessment

**User Story:** As a candidate, I want each of my answers to be evaluated by the AI, so that I receive a meaningful score and summary in the final report.

#### Acceptance Criteria

1. WHEN THE InterviewStateMachine enters `processing_answer`, THE InterviewHook SHALL call `POST /api/chat/ask` with a prompt asking the LLM to evaluate the candidate's answer and return a JSON object containing `score` (integer 1–10), `sentiment` (one of `positive`, `neutral`, or `negative`), `key_point` (a single sentence), and an optional `follow_up_hint` string.
2. WHEN the Groq API returns a valid AnswerAssessment JSON object, THE InterviewHook SHALL store the assessment in the Transcript entry for the current question and transition the InterviewStateMachine to `speaking_question` for the next question, or to `completed` if all 5 questions are answered.
3. THE VoiceInterviewModal SHALL NOT speak the AnswerAssessment aloud; assessments SHALL be displayed only in PhaseResults.
4. IF the Groq API call fails during answer assessment, THEN THE VoiceInterviewModal SHALL display an error notice and offer a "Retry assessment" button without discarding the transcribed answer.

---

### Requirement 8: Phase Intro UI

**User Story:** As a candidate, I want to see session details before the interview begins, so that I can prepare and grant microphone permission in advance.

#### Acceptance Criteria

1. WHILE THE InterviewStateMachine is in `intro`, THE VoiceInterviewModal SHALL render PhaseIntro displaying the job title, candidate name, number of questions (5), estimated duration (5 minutes), and a microphone permission warning.
2. THE PhaseIntro SHALL display a "Begin Interview" button that triggers question generation and transitions the InterviewStateMachine to `loading_questions`.

---

### Requirement 9: Phase Active UI

**User Story:** As a candidate, I want a clear, focused interview interface during the active session, so that I can concentrate on answering questions without distraction.

#### Acceptance Criteria

1. WHILE THE InterviewStateMachine is in any active interview state (`speaking_question`, `waiting_for_answer`, `recording`, `processing_answer`), THE VoiceInterviewModal SHALL render PhaseActive.
2. THE PhaseActive SHALL display a progress bar indicating the current question number out of 5 and the `focus_area` label of the current question.
3. THE PhaseActive SHALL display the AIAvatar component, which SHALL pulse with a CSS animation WHILE THE InterviewStateMachine is in `speaking_question`.
4. THE PhaseActive SHALL display a collapsible side panel listing all previously answered question-and-answer pairs.
5. WHILE THE InterviewStateMachine is in `processing_answer`, THE VoiceInterviewModal SHALL display a "Processing…" status label.

---

### Requirement 10: Phase Results UI

**User Story:** As a candidate, I want to see a comprehensive summary of my interview performance, so that I can understand how I did and take next steps.

#### Acceptance Criteria

1. WHEN THE InterviewStateMachine transitions to `completed`, THE VoiceInterviewModal SHALL render PhaseResults with a header reading "Interview Complete".
2. THE PhaseResults SHALL display the overall score as the arithmetic mean of all 5 per-question scores, rendered as a ring chart.
3. THE PhaseResults SHALL display a table with one row per question containing the question text, the candidate's answer transcript, the per-question score, and the `key_point` from the AnswerAssessment.
4. THE PhaseResults SHALL display a "Download Transcript" button that generates and downloads a plain-text file containing the full Transcript.
5. THE PhaseResults SHALL display a "Save to My Profile" button that calls `POST /api/candidate/interviews` with the complete interview payload.

---

### Requirement 11: Keyboard Shortcuts

**User Story:** As a candidate, I want keyboard shortcuts for common actions, so that I can control the interview without reaching for the mouse.

#### Acceptance Criteria

1. WHILE PhaseActive is rendered, THE VoiceInterviewModal SHALL toggle microphone recording when the Space key is pressed.
2. WHILE THE InterviewStateMachine is in `recording`, THE VoiceInterviewModal SHALL submit the current answer when the Enter key is pressed.
3. WHILE PhaseActive is rendered, THE VoiceInterviewModal SHALL display a pause modal when the Escape key is pressed.

---

### Requirement 12: Backend — Save Interview

**User Story:** As a candidate, I want my completed interview to be saved to my profile, so that recruiters and I can review it later.

#### Acceptance Criteria

1. THE Backend SHALL expose a `POST /api/candidate/interviews` route that accepts a JSON body containing `evaluation_id`, `questions`, `answers`, `assessments`, `overall_score`, `duration_seconds`, and `completed_at`.
2. WHEN a valid request is received at `POST /api/candidate/interviews`, THE Backend SHALL create a CandidateInterview record in the database with `candidate_id` derived from the authenticated user's session, and SHALL return the created record as JSON.
3. IF the authenticated user does not have a candidate profile, THEN THE Backend SHALL return HTTP 400 with an error message.
4. IF the `evaluation_id` does not correspond to an evaluation belonging to the authenticated candidate, THEN THE Backend SHALL return HTTP 403.
5. THE Backend SHALL expose a `GET /api/candidate/interviews/<eval_id>` route that returns the CandidateInterview record for the given evaluation if one exists, or HTTP 404 if none exists.
6. THE CandidateInterview SQLAlchemy model SHALL persist `id`, `candidate_id`, `evaluation_id`, `transcript` (JSON), `overall_score`, `completed_at`, and `duration_seconds` fields.

---

### Requirement 13: AI Closing Message

**User Story:** As a candidate, I want the AI to acknowledge the end of the interview verbally, so that I have a clear signal that the session is complete.

#### Acceptance Criteria

1. WHEN all 5 answers have been assessed and THE InterviewStateMachine is about to transition to `completed`, THE TextToSpeechHook SHALL speak a closing message such as "Thank you for completing the interview. Your responses have been recorded."
2. WHEN the closing message finishes speaking, THE InterviewStateMachine SHALL transition to `completed` and render PhaseResults.

---

### Requirement 14: Component File Structure

**User Story:** As a frontend developer, I want the interview feature organised into well-defined component and hook files, so that the codebase remains maintainable and each concern is isolated.

#### Acceptance Criteria

1. THE Frontend SHALL contain the following files:
   - `frontend/src/components/interview/VoiceInterviewModal.tsx`
   - `frontend/src/components/interview/InterviewPhaseIntro.tsx`
   - `frontend/src/components/interview/InterviewPhaseActive.tsx`
   - `frontend/src/components/interview/InterviewPhaseResults.tsx`
   - `frontend/src/components/interview/MicButton.tsx`
   - `frontend/src/components/interview/AIAvatar.tsx`
   - `frontend/src/hooks/useSpeechRecognition.ts`
   - `frontend/src/hooks/useTextToSpeech.ts`
   - `frontend/src/hooks/useInterview.ts`
2. THE InterviewHook SHALL own the InterviewStateMachine implemented with `useReducer` and SHALL expose state and dispatch to child components via props or context.
